# hayaosi - 対面クイズ大会用 早押し&選択回答システム

学校の友達同士（最大15人）で、対面のクイズ大会をスムーズに進行するためのWebアプリ。
早押しボタンとしても、4択クイズの回答端末としても使えます。

▶ **使う**: https://guttyanneruuuuuu.github.io/hayaosi/

## 機能

### 共通
- インストール不要、ブラウザだけで動く（スマホ・PC両対応）
- WebRTC データチャネルによる低遅延P2P通信
- 出題者端末で受信時刻を一元判定（公平な単一基準）
- 累計スコア機能（正解で +1pt）

### 早押しモード
- 大きな赤い早押しボタン、押すと光る演出+ブザー音
- 出題者画面に「押した順」がリアルタイム表示
- 1位の人に出題者が **◯（正解）/ ✕（不正解）** を判定
  - 正解 → 全員に **「ピンポン」** 音、結果バナーが緑に
  - 不正解 → 全員に **「ブブー」** 音、結果バナーが赤に、その人は今の問題で押せなくなり次の人が押せる
- 「次の問題へ」ボタンで即リセット

### 4択モード（選択問題）
- 回答者は 1〜4 のボタンで回答（正解発表前なら変更可能）
- 出題者画面に集計バー＋誰が何番を選んだかをリアルタイム表示
- 出題者が「正解は◯番」を発表 → 全員に音、選択肢が緑/赤で表示

### スコア・参加者管理
- 全員の累計スコアを出題者画面に表示（高得点順）
- 各回答者の自分のスコアもヘッダーに表示
- 「スコアをリセット」ボタンで0に戻せる

## 使い方

1. **出題者** が名前と合言葉（例：`3年A組`）を入れて「**部屋をつくる**」をタップ
2. **回答者全員** が同じ合言葉で「**部屋に入る**」をタップ（最大14人）
3. 出題者がモード（**早押し / 4択**）を切り替え
4. 現実で問題を出題
   - 早押しモード: 早押し → 出題者が◯×判定
   - 4択モード: 全員回答 → 出題者が正解発表
5. 「**次の問題へ ▶**」で次の出題に進む

## 技術構成

| 項目 | 採用技術 |
|---|---|
| 通信 | **PeerJS** (WebRTC データチャネル) |
| 音再生 | **WebAudio API**（AudioBuffer事前デコード + 即時再生） |
| 公開 | **GitHub Pages**（静的ホスティング） |
| 共通基準 | 出題者端末の `performance.now()` 受信時刻 |

### 公平性の仕組み
- 各クライアントの押下イベントは即時にWebRTCでホストへ送られる
- ホスト側 `performance.now()` の受信時刻で順位確定 → 機械の早押しボタンと同じ「単一の判定基準」
- ネットワーク遅延差は出るが、これは現実の早押しボタンでも同じ（むしろ遅延が可視化される分公平）

## 注意

- 出題者がブラウザを閉じると部屋は消えます（出題者がサーバー役のため）
- 同じ合言葉の部屋を同時に2つは作れません
- 学校WiFi等の一部ネットワークではWebRTCが遮断される可能性があります
- 安定運用には Wi-Fi での参加を推奨
